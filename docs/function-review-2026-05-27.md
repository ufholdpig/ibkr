## 二、Paper 训练场视角 — 本轮修改回顾

### 2.1 旧配置在 Paper 中的实际表现 (05/26 运行)

修改前 F 被绑定在 `bounce_sell` 下，Paper 账户为空头持仓。系统行为：

| 步骤 | 结果 | 日志 |
|------|------|------|
| bounce_sell 触发 SELL 信号 | ✅ 信号成功生成 | `F 反弹卖出: 市价 $14.94` |
| qty=-1 查询持仓 | ✅ 正确获取空头持仓 | `positions_map[F] = -5133` |
| 替换为实际数量 | ❌ 原代码只处理 SELL + qty>0 | 空头被忽略，quantity 未被替换 |
| 提交订单 | ⚠️ 提交时 qty=-1 无效 | 订单异常 |

**暴露的问题**：SELL 的 `-1` 替换逻辑没有处理空头持仓场景。这是管线代码的漏洞。

### 2.2 本轮修改内容

| 修改 | 文件 | 目的 |
|------|------|------|
| BUY 分支支持 `quantity="ALL"` → `-1` sentinel | `strategy.py:604-612` | 空头平仓模版（short_take_profit / short_stop_loss）需要 BUY + ALL 语义 |
| 持仓获取条件扩展为 SELL 或 qty=-1 | `put_order.py:252` | BUY(ALL) 信号也需要获取持仓数据 |
| BUY(-1) 预检查：无空头持仓则跳过 | `put_order.py:283-302` | 避免无效提交 |
| BUY(-1) 替换为空头持仓数量 | `put_order.py:310-312` | 空头平仓时 quantity = abs(negative_position) |
| 模版配置重构 | `config/ibkr.yaml` | bounce_sell 拆分为 long_take_profit / short_take_profit / short_stop_loss |
| stop_loss → long_stop_loss | 重命名 | 命名统一 |

### 2.3 测试要点

下一轮 Paper 运行应关注：

1. F 的 short_take_profit 信号能否正确生成 BUY(ALL)
2. BUY(-1) 能否被正确替换为空头持仓量
3. 空头无持仓时是否正确跳过
4. long_take_profit 对多头标的（DLR, META, MSFT, AMZN）是否能正常触发 SELL(ALL)

## 三、从 F 案例看策略模版改进方向

### 3.1 模版多样性不足

F 在 $13→$16 的反弹中，不同阶段需要不同的策略应对：

| 价格区间 | 合适的策略 | 当前是否有对应模版 |
|----------|-----------|------------------|
| $13（超卖） | 不建议追加（基本面差），或仅小仓抄底 | dip_buy ✅（但有基本面无过滤） |
| $13→$16（反弹中） | 持有或分批止盈 | long_take_profit ✅（但仅 ALL 清仓） |
| $16+（突破 52 周高） | 趋势跟踪或持有 | trend_entry ✅ |
| 任何价格（急跌） | 止损 | long_stop_loss ✅ |
| 任何价格（基本面恶化） | 减仓或清仓 | ❌ 无 |

**不足**：
- 没有"分批止盈"能力（当前所有模版都是 ALL 或固定数量）
- 没有"到达目标价后持有"的策略（如 trailing stop）
- 没有基本面驱动的信号

### 3.2 F 对模版参数的启示

| 指标 | F 值 | 对模版设计的参考 |
|------|------|----------------|
| Beta 1.66 | 高波动 | 止损阈值应放宽（如 -20% 而非 -15%），避免被正常波动震出 |
| 股息 4.54% | 高股息 | 长期持有有现金回报，止损触发意愿应降低 |
| EPS -$1.55 | 亏损 | 不适合价值投资类策略（dip_buy），适合趋势/动量策略 |
| 52周突破 | 强势 | trend_entry 可能在此类标的上找到机会 |

### 3.3 训练方向

| 方向 | 说明 | 验证方式 |
|------|------|---------|
| 策略参数调优 | 不同标的的止损比例、RSI 阈值、均线周期应不同 | Paper 运行积累信号/订单数据，分析胜率 |
| 模版组合效果 | 同一标的同时绑定多个模版的协同效果 | 观察冲突解决、优先级排序是否合理 |
| 市场环境适应性 | 同一模版在不同市场阶段（震荡/趋势/极端波动）的表现 | 跟踪 market_regime 与信号的对应关系 |
| 信号质量评估 | 信号生成后的执行结果、滑点、成交率 | 分析 order 记录的 filled_qty / avg_price |

## 六、总结

F 是一个有代表性的训练案例：低单价、高波动、基本面偏弱、近期有题材驱动。在 Paper 中训练策略时，F 这类标的能帮助验证：

- 高波动标的的止损/止盈参数是否合理
- 方向性模版（short 模版）是否能正确工作
- 多模版冲突解决是否有效

本轮修改完善了**信号管线中的空头平仓路径**。接下来应聚焦 P0 脚手架修复，然后进入持续的 Paper 运行→数据积累→策略迭代的循环，直到模版表现稳定到可以落地实盘。
