## 一、背景

原 `trend_entry.yaml` 由 6 个条件 AND 组成：

```
ma_stack(>) AND sma_slope(>5°) AND ma_spread(<5%) AND volume_spike(>1.3x) AND rsi(>45) AND (consolidation_breakout OR retrace_breakout)
```

自评报告确认：**全市场 350+ 只股票，0 只完全匹配 6/6 条件**。核心瓶颈：
1. `ma_spread < 5%` — 过滤 82% 候选（强势股 MA50 已远高于 MA200）
2. `consolidation/retrace ±3%` — 过滤 100% 最终候选（强势股价格不回到 MA50）
3. `sma_slope > 5°` — 横盘市中均线走平，极少达标

老大基于 PineScript 回测，给出了两轮参数优化方案。

### 2.3 策略哲学

老大在 05/26 回应中同步阐述了这个策略的设计思想：

> 这个策略是要找出长期是上行趋势的股票，但是中期要有相当大或者相当长时间的充分休整，到50天线附近才会触发再次入场。目前大盘处于高位，强势股已经涨了很多，继续追就很不安全，这个系统肯定是不会触发的。找不出太多股票是正常的，现在大背景就不是进场的时候。前面给的参数相对来说比较严格，参数可以先放松一些，慢慢再收紧。如果大盘出现5-10%回撤，估计到时候会扫除一大堆。这3个模版的监控列表。

> CRDO今天往上突破，后面如果继续离开当前区间确实是突破买入点。

### 2.4 解读

- **大背景决定可用标的数量**：当前大盘高位、强势股已大幅上涨，策略不触发是正常的系统行为，不是策略设计问题
- **参数采用"先松后紧"策略**：先大幅放松参数扩大覆盖，待大盘回撤出现更多候选后再慢慢收紧
- **CRDO 是近期关注标的**：今日向上突破，若持续离开盘整区间，将符合 consolidation_breakout 条件
- **三模板并联形成监控列表**：`trend_entry`(原版) + `trend_entry.1`(宽松) + `trend_entry.2`(优化) 同时运行，覆盖不同松紧度

## 五、推荐实施路线

### Step 1：纯 YAML 调参（5 行改动，立即生效）

```yaml
conditions:
  operator: AND
  rules:
    - type: ma_stack
      operator: ">"

    - type: sma_slope
      period: 50
      threshold: 0          # 5 → 0，允许任意正斜率
      multiplier: 60         # 45 → 60，对应 ~20%/5bar
      operator: ">"

    - type: sma_slope        # ★ 新增: MA200 必须缓慢上升
      period: 200
      threshold: 0
      multiplier: 10
      operator: ">"

    - type: ma_spread
      operator: "<"
      threshold: 0.30        # 0.05 → 0.30

    - type: volume_spike
      multiplier: 0.6        # 1.3 → 0.6

    - type: rsi
      operator: ">"
      threshold: 40          # 45 → 40

    - operator: OR
      rules:
        - type: consolidation_breakout
        - type: retrace_breakout
```

### Step 2：硬编码阈值修改（2 行，`market_data.py`）

```
compute_consolidation(..., threshold_pct=5.0)   # 3.0 → 5.0
retrace_to_ma50 = deviation_pct <= 5.0          # 3.0 → 5.0
```

### Step 3：打通 retrace_breakout 参数（~5 行，`retrace_breakout.py`）

让 YAML 中 `threshold: 5.0` 实际生效，替代硬编码。

1. **老大回测确认了自评报告的判断**: 原参数在横盘市中过于严格，0/350 通过率。v1/v2 虽大幅放松参数，但在当前牛市高位市场仅从 0 提升到 3 只触发
2. **最大瓶颈不是参数阈值，而是价格位置**: ma_spread 从 5% 放松到 30%、volume 从 1.3x 降到 0.6x 后，绝大多数 AI/强势股可通过这些条件；但 `consolidation/retrace` 要求价格靠近 MA50，强势股普遍偏离 20-50%，这**不是调参能解决的问题**
3. **v1 与 v2 在当前市况下等价**: 两模版触发完全相同的 3 只标的。v2 的 fib±5 容差仅在 7/8 分段体现优势，但无法跨越"价格偏离 MA50 太远"这一硬瓶颈
4. **v2 的 fib 滚动窗口设计在中分段更优**: 以 BAC 为例（cons_days=15），v1(fib±2) 无法匹配任何斐波那契数，v2(fib±5) 可通过；这印证了老大 .2 优化版的改进方向正确
5. **建议维持三模板并联，等待大盘回撤**: 当前阶段 v1/v2 的差异没有实际意义——因为触发标的太少。如老大所言，"如果大盘出现 5-10% 回撤，估计到时候会扫除一大堆"。届时 v2 的宽松 fib 容差将发挥更大作用
6. **Step 1~3 调参已完成**: YAML 参数调整、硬编码阈值修改均已体现在 v1/v2 模板中。Step 4+ 的斐波那契滚动窗口、双层 TP 等改动优先级降低——当前瓶颈不在这些功能，而在市场环境本身
